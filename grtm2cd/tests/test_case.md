# GRTM2CD ブラウザテスト手順書

**対象バージョン:** 0.7.2  
**実施環境:** ブラウザ（Chrome / Edge / Firefox）  
**前提:** index.html をブラウザで開いていること

---

## A. エンコード正常系

### A-1 基本エンコード

1. Encode タブを開く
2. 任意のファイル（例: test.pdf）を Treasure Map ゾーンにドロップ
3. 任意の画像（JPG/PNG）を Cat ゾーンにドロップ
4. 任意の画像（JPG/PNG）を Dog ゾーンにドロップ
5. **確認:** Cat PNG・Dog PNG の 2 ファイルがダウンロードされる

結果: [ OK ]
備考:

### A-2 JPG キャリア

- A-1 の Cat/Dog に JPEG ファイルを使用する
- **確認:** 正常にエンコードされ PNG が生成される

結果: [ OK ]
備考:

### A-3 WebP キャリア

- A-1 の Cat/Dog に WebP ファイルを使用する
- **確認:** 正常にエンコードされ PNG が生成される

結果: [ OK ]
備考:

### A-4 大きいキャリア画像（ダウンスケール）

- Cat に 4000×3000 px 以上の画像を使用する
- **確認:** ダウンスケールされたうえでエンコードが完了する

結果: [ OK ]
備考:

### A-5 大きい Treasure Map（解像度自動選択）

- サイズの大きいファイル（例: 1MB 超）を Treasure Map にドロップ
- **確認:** 必要な解像度が自動選択され、エンコードが完了する

結果: [ OK ]
備考:

## B. デコード正常系

### B-1 基本デコード

1. A-1 で生成した Cat PNG・Dog PNG を Decode タブにドロップ
2. **確認:** 元の Treasure Map ファイルが復元・ダウンロードされる
3. **確認:** ファイル名が元と一致している
4. **確認:** ファイル内容（バイナリ比較）が元と一致している

結果: [ OK ]
備考:

### B-2 入力順序を逆にしてデコード

- B-1 と同じファイルを **Dog を先・Cat を後** の順でドロップする
- **確認:** B-1 と同じ結果が得られる（Cat/Dog 自動判定）

結果: [ OK ]
備考:

### B-3 日本語ファイル名の Treasure Map

1. ファイル名が日本語のファイル（例: `宝の地図.pdf`）をエンコード
2. 生成した PNG をデコード
3. **確認:** 日本語ファイル名が正確に復元される

結果: [ OK ]
備考:

## C. エラー系

### C-1 空ファイル

1. サイズ 0 バイトのファイルを Treasure Map にドロップ
2. **確認:** エラーメッセージが表示され、処理が止まる

結果: [ OK ]
備考:

### C-2 無関係な画像ペアのデコード

1. エンコードに使っていない任意の 2 枚の画像を Decode タブにドロップ
2. **確認:** ペア不一致を示すエラーメッセージが表示される

結果: [ OK ]
備考:

### C-3 再保存・再圧縮された PNG のデコード

1. A-1 で生成した Cat PNG を画像編集ソフト（ペイント等）で開き、上書き保存する
2. 元の Dog PNG と組み合わせて Decode タブにドロップ
3. **確認:** 復号失敗を示すエラーメッセージが表示される（LSB 破壊）

結果: [ OK ]
備考:
